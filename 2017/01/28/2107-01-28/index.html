前言

在这里节选一部分你不知道的JavaScript，一段进行理解。

第一稿（有问题）

对我来讲这里为什么用apply方法是让我最费解的地方，那么我假设如果我需要往传入的模块function，即impl参数传入依赖模块参数，然而我想了想，好像没有这样的方法，想要传入参数只能靠apply和call的方法，而call的方法传入的不是数组，所以在这里apply可能会更好点。那么这就涉及到，this应该指向谁呢？

根据高程里面，apply的用法实际上是为了指定函数方法里面的this的指向。也就是说，如果函数里面有this，那么指向就会被动态的改变。比如Function.apply(window)指向的是window对象，详见高程117页的例子。而如果传入的函数没有this，也就没有什么作用。

在impl.apply(impl, deps)里，也就是将this指向了impl函数本身，并且将依赖模块作为参数传入了impl函数，可以被调用。这也就意味着作用域在impl函数中，而impl函数是modules对象的方法，即作用域是modules对象。

总结：impl.apply(impl, deps)这一核心代码，就是为了将依赖传入定义模块中，并且指定作用域为modules对象本身。

第二稿

抱着严谨的态度重新看了一遍，结果发现，错的有点离谱。这个写法其实还限定了如何定义模块的写法。如果模块的写法有问题，也会导致模块出现问题。

现在来看看如何写定义模块的：

可以看到模块的定义是有格式的，必须返回一个对象。也就是将模块内的方法封装到了一个对象里，这样子impl.apply(impl, deps)其实是进行了一次调用，返回了{hello : hello};对象，而且this指向了这一对像上。并非第一稿的理解不够全面。

这也是书上为什么说包装函数始终就是包装函数，对封装模块没有什么改变，因为它只是让封装变得更容易看而已，定义格式仍然是死的。
