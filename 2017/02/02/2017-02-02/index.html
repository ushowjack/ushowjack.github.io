原型链的关系

2017-02-02


通过学习数据类型（引用类型和基本类型）、闭包（closure）、还有this的用法（关于动态指向）、以及对象的一些属性，比如最重要的的是：

对于构造器（constructor）和原型（prototype）的理解。

每一个原型的属性里有一个默认属性：constructor。通过这个属性，原型可以指向自己的构造器，然而个人觉得这个知识点并没有什么卵用。

每一个构造器都会有一个默认属性：prototype。通过这个属性，构造器就可以指向自己的原型了。

那么什么是构造器呢？构造器本质是一种函数，通过this来设置函数的属性和方法。最后通过new的方法来创造出一个实例。而实例的本质就是对象。

也就是说构造器就相当于一个模具，通过new来创造出一模一样的对象。

那么这些实例、构造器和原型之间有什么联系吗？

  1. 构造器与原型之间：它们是互相指向的。构造器实际上是一个伪类（也就是一个中介），原型就像是房东，需要中介（构造器）来替原型产生出新的租客（实例),并且打上原型的标签。一旦创造出来实例，那么构造器也就和实例、原型没有什么关系。当然也可以继续为原来的原型继续服务。

  2. 实例和构造器之间：通过构造器，被创造出来的实例会自动有一个默认标签proto（默认属性），通过这个属性指向原型（有点像签了合同认了房东）。这个时候，实例对象被创建出来以后就和构造器没有半毛钱关系了，当构造器重新指向新的原型，或者添加、删除、更改属性方法，都不会影响实例的属性方法。

  3. 实例和原型之间：原型相当于实例的水厂（属性、方法），当水箱（实例）里没有水（属性、方法）了，那么就会从水厂（原型）里调用水（属性、方法）。
  4. 实例与实例之间：它们之间的关联只有一个，那就是proto都是指向共同的原型。

总结

而在面向对象里面最重要的是要理解，实例与原型之间的关系，也就是水箱和水厂的关系，通过这个关系可以看出来，当实例中没有某一变量的时候，就会向上调用原型的变量。而当实例覆写了变量之后，就会把原型的变量覆盖。
